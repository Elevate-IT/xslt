# XSLT 1.0 + script to XSLT 3.0 migration guide

## Objective
Remove all embedded script blocks and Microsoft-specific XSLT extensions so transformations run on a pure XSLT 3.0 processor (for example Saxon 12).

## Current state
Use the audit script to generate the exact inventory:

PowerShell command:

    ./tools/xslt3-migration-audit.ps1

Artifacts:
- xsl-out/xslt-v3-migration-audit.csv
- xsl-out/xslt-v3-migration-summary.txt

## Migration strategy
1. Inventory and classify.
2. Replace common script calls with native XPath/XSLT functions.
3. Refactor stateful logic (counters, static lists) into XSLT 3.0 grouping, accumulators, and maps.
4. Remove msxsl namespaces and script blocks.
5. Set stylesheet version to 3.0 and validate output parity with baseline samples.

## Common replacements

### String/date helpers
- MyScript:RemEmptySpace(x):
  - If intent is trim + collapse spaces: normalize-space(x)
  - If intent is remove all whitespace: replace(string(x), '\\s+', '')
- MyScript:ToUpper(x): upper-case(string(x))
- MyScript:TrimEnd(x): replace(string(x), '\\s+$', '')
- MyScript:Replace(x, a, b): replace(string(x), a, b)
- MyScript:Abs(x): abs(xs:decimal(x))
- MyScript:ParseDate(...): implement a dedicated xsl:function for each known input format and return with format-date()/format-dateTime()

Example (yyyyMMdd -> yyyy-MM-dd):

    <xsl:function name="local:parse-yyyymmdd" as="xs:string?">
      <xsl:param name="s" as="xs:string?"/>
      <xsl:sequence select="
        if (empty($s) or normalize-space($s) = '') then ()
        else format-date(xs:date(concat(substring($s,1,4), '-', substring($s,5,2), '-', substring($s,7,2))), '[Y0001]-[M01]-[D01]')
      "/>
    </xsl:function>

### Counters and line numbers
- MyScript:GetLinCounter / GetNRCounter:
  - Use position() inside for-each.
  - For document-wide counters, use xsl:number or xsl:accumulator.

### Stateful aggregations
- userCSharp:AddAccount + GetAccountAmount/GetVATAmount:
  - Replace with xsl:for-each-group group-by and sum().
  - If random lookups are required, build map(*) via xsl:map and map:get().

### GUID and current timestamps
- MyScript:GetCurrentDate:
  - Use current-date()/current-dateTime() with format-date()/format-dateTime().
- MyScript:GetGUID:
  - Prefer passing GUID as an external parameter generated by the caller.
  - If unavoidable, create deterministic IDs with generate-id() plus timestamp, but this is not a true UUID.

### Blocking operations
- MyScript:Sleep:
  - Remove from XSLT. Delays should be handled by orchestration/workflow code, not transformation logic.

## XSLT 3.0 baseline template

    <xsl:stylesheet version="3.0"
      xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
      xmlns:xs="http://www.w3.org/2001/XMLSchema"
      xmlns:local="urn:local"
      expand-text="yes"
      exclude-result-prefixes="xs local">

      <xsl:mode on-no-match="shallow-copy"/>

      <!-- local:xsl-functions here -->

      <xsl:template match="/">
        <xsl:next-match/>
      </xsl:template>
    </xsl:stylesheet>

## Suggested rollout
1. Start with one customer folder and migrate 3 to 5 representative files.
2. Build a small regression set: same input, compare canonicalized XML output.
3. Promote proven helper functions into shared includes/imports.
4. Batch-migrate remaining files by function family (ParseDate, counters, aggregation, etc.).

## Definition of done per file
- No msxsl:script or xsl:script blocks.
- No MyScript:* or userCSharp:* calls.
- No dependency on urn:schemas-microsoft-com:xslt.
- version="3.0" in stylesheet.
- Output equivalent for approved test payloads.
